# User Stories — expo-web-unify

Small consolidation feature. ~2 h total. No JTBD (motivation self-evident: "stop maintaining two web apps").

---

## US-01: Firebase Hosting serves the Expo web bundle

**As** Clemens
**I want** `firebase.json` and the deploy workflow to build and publish the Expo web export (`npx expo export -p web` → `dist/`)
**So that** a push to main publishes the real RN UI, not the stripped-down Vite SPA.

**Journey steps**: swap-config, swap-workflow, ci-and-deploy, verify

### Acceptance Criteria

```gherkin
Scenario: firebase.json points at the Expo web output path
  Given firebase.json at repo root
  Then hosting.public is "dist"
  And the SPA rewrite rule still maps "**" to "/index.html"

Scenario: deploy-web.yml runs the Expo web export
  Given .github/workflows/deploy-web.yml
  Then one step runs `npx expo export -p web` at the repo root (not inside /web)
  And that step runs before the Firebase Hosting deploy step
  And no step runs `npm ci` or `npm run build` inside /web

Scenario: Live URL serves the Expo web bundle
  Given a successful deploy after migration
  When I GET https://grocery-list-cad.web.app
  Then the response is 200
  And the body is the index.html generated by `expo export -p web` (detectable by a characteristic marker, e.g., the `<title>` matches the Expo app's title or an Expo-bundled script path is referenced)
```

## US-02: Retire the `/web` Vite app

**As** Clemens
**I want** the `/web` directory and its tests deleted, and all references removed
**So that** nobody (me, future-me, claude) accidentally edits a dead codebase or reinstalls its dependencies.

**Journey steps**: delete-vite

### Acceptance Criteria

```gherkin
Scenario: /web directory is gone
  Given the repo after migration
  Then /web does not exist
  And grep -r "web/src" in production files (excluding docs/) returns no hits
  And grep -r "web/package" in production files returns no hits

Scenario: Root workspace still builds
  When I run `npm install` at repo root
  Then it succeeds without referring to /web
  And `npm test` (main repo jest) passes
  And `npx expo export -p web` produces a dist/ directory

Scenario: No stale deploy triggers
  Given .github/workflows/deploy-web.yml after this feature
  Then no step depends on /web existing
  And CI passes without /web
```

## US-03 (optional salvage): Hoist validation + error-mapping to shared mobile auth

**As** Clemens
**I want** the pure `validateFormInput` + `mapAuthError` logic available in `src/auth/`
**So that** the mobile `LoginScreen.tsx` can consume shared rules rather than keeping them inline.

Skip this story if effort exceeds appetite — mobile currently works with inline rules. Marked optional.

**Journey steps**: (post-migration cleanup)

### Acceptance Criteria

```gherkin
Scenario: Shared validation module
  Given src/auth/validation.ts
  Then it exports validateFormInput + EMAIL_PATTERN + AuthMode
  And the content matches /web/src/auth/validation.ts (pre-deletion)

Scenario: Shared error-mapping module
  Given src/auth/error-mapping.ts
  Then it exports mapAuthError(error, mode) with the same mode-aware copy

Scenario: Mobile LoginScreen uses the shared modules
  Given src/ui/LoginScreen.tsx after this story
  Then it imports validateFormInput and mapAuthError from ../auth/
  And its inline copies of those rules are removed
  And all existing LoginScreen acceptance tests pass unchanged
```

## Traceability

| Story | Journey steps | Priority | Effort |
|---|---|---|---|
| US-01 | swap-config, swap-workflow, ci-and-deploy, verify | must | 30 min |
| US-02 | delete-vite | must | 15 min |
| US-03 | (optional cleanup) | nice-to-have | 30 min |

**Critical path**: US-01 + US-02 together. Neither is shippable without the other — you can't delete `/web` while the deploy pipeline points at it, and you don't want to point the pipeline at an Expo bundle while `/web` still exists as a source of confusion. Ship them as one atomic delivery.
